# Plugin Pages 碎镜视觉规范

本文记录 `ncqq-dashboard` 一级菜单的碎镜构图规则，用于后续维护时避免退回普通卡片、外圈碎屑或松散布局。

## 参考样式识别

参考图属于整屏破镜拼图式菜单，不是卡片列表。画面主体由多个不规则玻璃碎片围绕中心裂点展开，碎片之间间距很窄，整体覆盖画面中部的大部分宽高，只在四周留少量暗色背景。

关键视觉特征：

- 构图中心在画面中部略偏右，所有主碎片像从该中心爆裂出去。
- 主碎片约 6 到 9 个，大小不完全一致，但都属于同一组破镜面，不应像独立卡片散落。
- 最大碎片分布在左中、右上、右中和下中区域，避免底部或右侧出现大块空白。
- 碎片形状以锐角多边形为主，边缘不圆角，局部有斜切和尖角。
- 碎片边缘有细白高光和轻微外发光，暗部阴影很薄，不使用厚重外框。
- 第一阶段镜片形态和光边由生成图资产提供，避免 CSS 多边形显得生硬。
- 第一阶段使用生成图作为主镜片视觉层，不渲染外圈小碎屑。
- 菜单文字贴在碎片内部，带小徽章或短标签；文本服务于识别，不抢占碎片画面。
- 碎镜模式不得露出整屏背景看板娘。
- 不额外渲染补位碎片或背景碎屑，避免画面变脏。

## Dashboard 映射

- manager 入口按节点顺序占用非固定碎片。
- 审批入口固定上方暗色碎片，绑定关系入口固定下方暗色碎片，避免与 manager 节点争抢主视觉碎片。
- 真实入口不足时只补透明热区位置，不额外绘制视觉碎片。
- 一级菜单不再保留三排直出、横向瀑布流、A/C 背景试验、星河流动遮罩或旧 `shards` 分支。
- 二级弹窗交互不参与一级碎镜构图，点击真实碎片后仍打开对应详情面板。

## 资源拆分

- 原始破镜整图保留在 `pages/ncqq-dashboard/assets/memory-shards-generated-v2.png`，用于追溯生成结果和重新裁切。
- 基础单片镜片资源存放在 `pages/ncqq-dashboard/assets/memory-shards/`，文件名按视觉位置命名，例如 `shard-01-upper-left.png`。
- 当前页面运行层使用 `pages/ncqq-dashboard/assets/memory-shards-scene/*-scene-v14.png`，完整玻璃底图保留，只裁已确认溢出的看板娘贴图区域，不做全局内缩。
- `pages/ncqq-dashboard/assets/memory-shards/manifest.json` 记录基础单片的源图裁切框、多边形点位、自然尺寸和相对源图的百分比位置。
- `pages/ncqq-dashboard/assets/memory-shards-scene/manifest.json` 记录当前运行版内景镜片的来源和裁切策略。
- 告警边缘光使用 `pages/ncqq-dashboard/assets/memory-shards-alert/*-alert-*-v17.png`，审批黄光避开顶中碎片右侧错误高光，只保留底部短边提示。
- 一级背景使用 `pages/ncqq-dashboard/assets/backgrounds/sora-shattered-star-bg-v7-generated-fill.png`，不得恢复已删除的 v3/v4/v5 背景候选。
- 后续如需要做独立动效、悬停高亮或单片替换，应优先读取 manifest 复用已有点位，不重新引入瀑布流、规整卡片网格或背景碎屑。

## 实现约束

- `stage.css` 只负责一级 memory 镜片视觉层和透明热区位置，不写业务逻辑。
- 碎镜模式以 `.memory-board` 为边界，旧 `tiles`、`stream`、`shards` 分支不再参与一级菜单。
- 主碎片使用绝对定位，坐标必须围绕中心裂点，避免水平排布、规整网格或大面积留白。
- hover 只显示节点名称纯文字，不使用卡片、面板或大块遮罩。
- manager 下存在离线实例时，可在对应碎片靠近中心的一侧边缘叠加红色半边光；提示光必须从实际镜片 alpha 边界提取并限制在碎片内侧窄边缘带。
- 审批有待处理项时使用黄色半边边缘光；绑定关系入口不显示状态光。
- 小屏下可以降级为两列网格，并隐藏不可点击补片。
- 修改本视觉规则时同步 `docs/plugin-pages-architecture.md`。
